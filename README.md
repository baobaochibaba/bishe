# 基于嗅碳卫星碳排放检测的综合能源系统低碳调度研究

> **毕业设计项目**  
> 利用 OCO-3 卫星 XCO₂ 观测数据与 ERA5 气象数据，反演点源碳排放，构建含动态碳信号与阶梯碳价的综合能源系统优化调度模型。

---

## 项目简介

本项目主要完成以下两部分工作：

1. **碳排放反演**  
   - 处理 OCO-3 卫星的 XCO₂ 数据，结合 ERA5 再分析气象数据（风速、边界层高度、总云量）。  
   - 通过高斯烟羽模型反演出目标点源（如电厂）的小时级碳排放强度。  
   - 生成典型日 24 小时排放通量序列，作为后续调度模型的动态碳信号。

2. **综合能源系统低碳优化调度**  
   - 构建含 CHP（热电联产）、燃气锅炉、电/热储能、光伏、需求响应的 IES 模型。  
   - 引入动态电网排放因子与阶梯碳价机制，建立两阶段优化模型：  
     - **第一阶段**：最小化总运行成本（含购电、燃气、储能、需求响应、碳成本）。  
     - **第二阶段**：在成本接近最优的前提下，最小化购电峰谷差、购电波动及锅炉峰值。  
   - 使用 YALMIP + CPLEX 求解器进行求解，并设置 4 种场景进行对比分析（基准、仅储能、仅需求响应、储能+需求响应）。

## 文件结构
├── test.m # 卫星数据反演主脚本（高斯烟羽反演 + 典型日排放生成）
├── shujv.m # 生成调度模型输入文件 dispatch_input.mat
├── main_dispatch.m # 综合能源系统两阶段优化调度模型（函数）
├── run_cases.m # 批量场景运行与对比分析脚本
├── README.md # 本文件
├── carbon+DR数据.xlsx # 负荷、电价、光伏数据（需自备）

└── dispatch_input.mat # 调度输入数据（由 shujv.m 生成）

## 运行环境

- **MATLAB R2020b** 或更高版本  
- **YALMIP** 工具箱（用于优化建模）  
- **CPLEX** 求解器（需安装并配置 MATLAB 接口）  
- 无额外工具箱要求（基本 MATLAB 即可运行）

> **注意**：脚本内部的数据文件路径为绝对路径，运行前请根据本机实际路径修改。

## 使用步骤

### 1. 数据准备

#### ① OCO-3 卫星数据（.nc4 文件）
- 来源：[NASA GES DISC](https://disc.gsfc.nasa.gov/)  
- 数据集名称：  
  `OCO-3 Level 2 bias-corrected XCO2 and other select fields from the full-physics retrieval aggregated as daily files, Retrospective processing V11r (OCO3_L2_Lite_FP 11r)`  
- 下载时请筛选研究区域的经纬度及研究日期范围，将下载的 `.nc4` 文件放入同一文件夹，并修改 `test.m` 中的 `dataFolder` 路径。
- 下载流程可参考该视频https://www.bilibili.com/video/BV1Mv411t7r6?vd_source=342f65127c2bf00be528cf1d10502674
本设计选取的经纬度范围为107.6-109.5/33.7-34.75，时间为2021年11月14日至2022年3月15日
#### ② ERA5 气象数据（.nc 文件）
- 来源：[Copernicus Climate Data Store](https://cds.climate.copernicus.eu/)  
- 所需变量：  
  - 10 米风速 u 分量 (`u10`)  
  - 10 米风速 v 分量 (`v10`)  
  - 边界层高度 (`blh`)  
  - 总云量 (`tcc`)  
- 只需下载目标典型日的时间段，并裁剪到研究区域经纬度。（本设计最终选择日期为2022-02-24日目标经纬度的气象数据，经纬度同上oco3卫星数据）  
- 修改 `test.m` 中 `era5_file` 的路径。
-下载流程可参考该视频【ERA5气象数据数据下载】https://www.bilibili.com/video/BV1MU3vzoEuK?vd_source=342f65127c2bf00be528cf1d10502674
#### ③ 负荷与电价数据
- `carbon+DR数据.xlsx` 需包含以下行：  
  - 第 1 行：24 小时电负荷（kW）  
  - 第 2 行：24 小时热负荷（kW）  
  - 第 3 行：24 小时电价（元/kWh）  
  - 第 4 行：24 小时光伏可用出力（kW）  
- 确保文件位于项目根目录，或在 `test.m` / `shujv.m` 中修改对应路径。

### 2. 修改脚本路径

打开 `test.m` 和 `shujv.m`，将以下变量改为本机实际路径：
- `dataFolder`（OCO-3 数据文件夹）  
- `era5_file`（ERA5 数据文件）  
- `loadFile`（若 Excel 文件不在根目录）

### 3. 运行反演

在 MATLAB 中运行：
test.m
该脚本会：
读取并筛选 OCO-3 观测数据
匹配 ERA5 气象数据
执行高斯烟羽反演
生成典型日 24h 排放序列，保存为 typical_day_one.mat

4. 生成调度输入
运行：
shujv.m
该脚本会读取反演结果及负荷数据，生成 dispatch_input.mat，供优化模型使用。

5. 运行调度模型与场景对比
运行：
run_cases.m
该脚本会：
依次调用 main_dispatch.m 求解 4 种场景
在命令行输出各场景成本、碳排放等指标
绘制多张对比图（成本构成、碳排放、购电曲线、储能荷电状态等）
核心算法说明
高斯烟羽反演模型
位于 test.m 底部的子函数 gaussian_plume_inversion_full：
将经纬度坐标转换为以源点为原点的局地坐标系，并根据气象风向旋转到下风向。
利用 Briggs 扩散系数（根据 Pasquill 稳定度等级选择）计算烟羽横向与垂向扩散参数。
由下风向观测的 XCO₂ 增强值反推源排放强度，并考虑混合层高度对烟羽的抑制效应。
取各观测点估计值的中位数作为最终排放通量。
大气稳定度分类
子函数 get_pasquill_stability 根据太阳高度角、云量和 10 米风速计算 Pasquill 稳定度等级（A–F），其中太阳高度角由天文算法给出。

两阶段优化模型
主函数 main_dispatch.m：
第一阶段目标：最小化包含购电、燃气、储能、需求响应及阶梯碳成本的总成本。
第二阶段目标：在总成本允许上浮 0.5% 的约束下，最小化购电峰谷差、相邻时段购电波动、锅炉峰值及锅炉出力波动。
约束条件包括功率平衡、设备出力限制、爬坡约束、储能荷电状态平衡及需求响应转移量守恒等。
所有核心算法均在代码内配有详细的行内注释，可直接阅读源码了解每一步的计算逻辑。
运行结果
run_cases.m 运行后将在 MATLAB 工作区生成 results 结构体数组和 ResultTable 汇总表，并自动绘制多张对比图，包括：
各场景总成本与成本构成
含碳/不含碳成本对比
总碳排放与碳排放来源
超额排放与阶梯碳段分配
购电曲线、锅炉供热、储能 SOC、需求响应量等细节
可根据需要自行保存图表。
注意事项
所有路径根据实际存放位置修改，建议使用相对路径或将数据统一放在项目根目录下的 data/ 文件夹中。
若 CPLEX 未安装或 YALMIP 未配置，运行调度模型时会报错，请确保求解器可用。
反演脚本对观测数据的质量筛选较为严格，若目标日无有效事件，程序会给出错误提示，请检查区域范围或数据时间覆盖。
生成的 .mat 文件下次可直接使用，无需重复运行前两步。
